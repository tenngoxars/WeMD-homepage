# 关于 WeMD

WeMD 的诞生源于对现有微信编辑器的三个不满：
1. **数据绑架**：大多数微信编辑器强制云同步，你的数据变成了它们的资产。
2. **排版割裂**：写作在 A 处，复制到公众号 B 处后样式全乱，需要二次调整。
3. **样式太丑**：有些在线微信编辑器的样式及其丑陋，让人不忍直视。

因此，WeMD 确立了两大核心原则：

## Local First
**Local First** 不仅仅是「离线可用」。它意味着：
- **数据所有权**：数据的主副本永远在你的本地硬盘上。
- **软件长存性**：假设 WeMD 的服务器关停了，只要你的电脑里有这个软件，你就可以永远使用它，且功能不受任何影响。
- **隐私**：你的文章草稿不需要经过任何人的服务器。

我们在技术上通过 [File System Access API](https://developer.mozilla.org/en-US/docs/Web/API/File_System_Access_API) 实现了这一点。浏览器直接读写你的硬盘，中间没有云端服务器参与。

## 只为公众号生态设计
其他的编辑器是「通用的」，而 WeMD 是「专用的」。
我们针对微信公众号的渲染引擎（基于 Webkit）进行了专门的 CSS 调优。
- **内联样式**：为了防止样式在粘贴时丢失，我们在渲染时就计算好了 Computed Styles。
- **兼容性**：我们测试了 iOS、Android 微信客户端的深色模式表现，确保你的文章在手机上看起来和电脑上一样好。
