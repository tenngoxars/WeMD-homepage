# 常见问题

---

## 使用问题

### 复制到公众号后样式丢失？

可能原因：
1. **使用了不支持的浏览器** — 推荐使用 Chrome 或 Edge
2. **公众号编辑器问题** — 尝试刷新公众号后台页面后再粘贴
3. **粘贴方式不对** — 确保使用 `Ctrl+V` / `Cmd+V`，不要右键粘贴

---

### 图片显示不出来？

检查以下几点：
1. **图片是否上传成功** — 看编辑器中图片链接是否完整
2. **图床配置是否正确** — 检查 Domain 是否以 `http://` 或 `https://` 开头
3. **跨域问题** — Web 版可能受限，建议用 Desktop 版测试

---

### 本地存储模式无法使用？

File System Access API 有浏览器限制：

| 浏览器 | 支持情况 |
| :--- | :--- |
| Chrome / Edge | ✅ 原生支持 |
| Safari | ❌ 不支持 |
| Firefox | ⚠️ 需在 `about:config` 开启 `dom.fs.enabled` |

---

### 文件没有自动保存？

- **Web 版**：使用浏览器存储模式时会自动保存；使用本地存储模式时也会自动保存
- **Desktop 版**：自动保存到本地文件夹

如果担心丢失，可以手动按 `Ctrl+S` / `Cmd+S` 保存。

## 主题问题

### 代码块没有语法高亮？

自定义主题 CSS 中**不要**给 `#wemd pre code.hljs` 设置 `color` 属性，否则会覆盖语法高亮颜色。

正确写法：
```css
#wemd pre code.hljs {
    background: #f5f5f5;
    /* 不要写 color: xxx; */
}
```

---

### 自定义主题不生效？

检查 CSS 选择器是否以 `#wemd` 开头：
- ✅ `#wemd p { ... }`
- ❌ `p { ... }`

---

## 图床问题

### 检查配置时报跨域错误？

Web 版受浏览器安全策略限制。解决方案：
1. 使用 **Desktop 桌面版**验证
2. 或在云服务商控制台开启 CORS（允许所有来源）

---

### AccessKey 安全吗？

密钥只保存在你的浏览器本地存储中，**不会**上传到任何服务器。

---

## 其他问题

### WeMD 会收集我的数据吗？

不会。WeMD 是 Local First 设计，所有数据都保存在本地。我们没有后端服务器存储用户数据。

---

### 可以离线使用吗？

写作和编辑功能可以离线使用，但**图床上传需要网络连接**（Web 版和 Desktop 版都一样）。
